### 任务一 : git常用命令学习

在这一部分中，你将学习如何操作使用git，git学习资料见[此文档](https://gitlab.buaaoo.top/oo_2020_public/guide_book_public/tree/master/git-guide)。首先，你需要按照step 1~step 6的提示完成每步对应的操作。操作完成之后，你将获得两个文件夹`poly`和`share`，其中`share`文件夹下的代码是用于学习接口技术的例程，`poly`文件夹下的代码是任务二需要的源代码。

#### step 1 新建仓库

在本地新建一个空文件夹， 在此目录下打开终端(bash/git bash/powershell/…)。

输入

> git init

你将看到类似这样的内容：

> Initialized empty Git repository in /Users/xxx/xxx/experiment1/.git/

这样我们得到了一个新的git仓库。

#### step 2 关联远程仓储

目前为止，我们的文件夹下还是一个空文件夹，需要关联两个远程仓储，我们先考虑第一个。

第一个是课程提供课程的远程仓储`share`

> git remote add share git@gitlab.buaaoo.top:oo_2020_public/exp_1_stu.git

#### step 3 从远程仓储拉取文件

可以从远程仓储拉取文件。 执行命令

> git pull share master

你将看到以下内容：

```

remote: Total 8 (delta 0), reused 0 (delta 0)
```

从`share`仓储的master分支下得到两个文件夹

#### step 4 忽略不必要的文件

文件夹`share`中的内容是第二个任务的例程，可以尝试跑通，文件夹`poly`中的内容是第二个任务中需要完善的程序，是需要提交的，而我们要求, 提交时**忽略** `share` 文件夹下的所有内容。 故我们要使用`.gitignore`

在当前工作的*根目录*下新建文件`.gitignore`

在其中写入

> share/*

表示push时忽略子目录`share`下的所有文件。

#### step 5 删除与远程仓储的关联

**为了避免不必要的干扰**，我们删除与远程仓储`share`的关联。

> git remote remove share

### 任务二：补全程序——多项式的排序

#### 任务概述

本任务提供了一份不完整的源代码（即任务一中你所拉取的`poly`文件夹中的内容），该源代码欲完成多项式的排序。首先，你需要阅读下面的要求，并参照share文件夹中的例程，用接口技术将代码补充完整。其次，你需要按照要求撰写实验报告。最后，请提交代码与报告。

#### 问题描述

给定一个多项式，不合并同类项，按照如下顺序，逐行逐一输出多项式的每一项:

1. 如果两个单项式的指数不同，指数高的在前面
2. 如果两个单项式的指数相同，系数小的在前面

#### 输入格式

若干行，每行两个数，表示这个多项式的每一项，第一个数为这一项的系数，第二个数为这一项的指数。(系数和指数的绝对值不超过100，多项式总项数不超过100000)

#### 输出格式

若干行，有序输出多项式的每一项，每一项按照如下格式输出（不妨简记系数为c*c*，指数为i*i*）：

取从上到下第一条符合的规则输出即可。

1. 若 c = 0*c*=0则输出 0
2. 若 c \ne 0*c*̸=0 且 i = 0*i*=0 则输出c (正整数符号无需输出)
3. 若 c = 1*c*=1 且 i \ne 1*i*̸=1 则输出 x^i (正整数符号无需输出)
4. 若 c = 1*c*=1 且 i = 1*i*=1 则输出 x
5. 若 c = -1*c*=−1 且 i \ne 1*i*̸=1 则输出 -x^i (正整数符号无需输出)
6. 若 c = -1*c*=−1 且 i = 1*i*=1 则输出 -x
7. 若 i = 1*i*=1 则输出c*x (正整数符号无需输出)
8. 不符合以上情形，按照c*x^i输出即可。(正整数符号无需输出)

#### 输入样例

```
2 4
0 3
1 4
-1 3
-1 1
1 1
3 0
2 1
3 -1
```

#### 输出样例

```
x^4
2*x^4
-x^3
0
-x
x
2*x
3
3*x^-1
```
